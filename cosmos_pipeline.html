01 — Contexte & Objectif

Automatiser les flux data d'une certification cosmétique

🌿
La problématique
Dans le secteur des cosmétiques biologiques et naturels, la certification COSMOS (standard européen) impose un suivi rigoureux des produits certifiés et des matières premières (RM) autorisées par chaque organisme certificateur (CB).

Chaque mois, les CB transmettent des fichiers Excel de référence déposés sur Google Drive. Jusqu'ici, la mise à jour des catalogues et la détection des changements de matières premières se faisaient manuellement, sans historique fiable ni alertes automatisées. La charge opérationnelle était élevée et les risques d'erreur importants.
🎯
La solution
Deux Cloud Functions GCP (Python 3.11) déclenchées automatiquement par Cloud Scheduler ingèrent les fichiers Drive, calculent les deltas, alimentent BigQuery et envoient des emails d'alerte SMTP. Looker Studio se rafraîchit automatiquement depuis BQ pour les visualisations.
⚡
L'impact
Passage d'un suivi 100 % manuel à un pipeline entièrement automatisé : détection immédiate des nouveaux ingrédients ADDED/REMOVED, historique complet dans BigQuery, alertes email aux parties prenantes, et dashboard Looker Studio toujours à jour — le tout dans les limites du free tier GCP.
02 — Pipeline Catalogue

Ingestion mensuelle du catalogue COSMOS

Catalogues COSMOS → BigQuery Cloud Scheduler · 16 de chaque mois à 9h UTC
Chaque mois, un fichier catalog-export-YYYY-MM-DD.xlsx est déposé sur Google Drive. La Cloud Function catalog_ingest détecte automatiquement les nouveaux fichiers non encore ingérés (idempotence via BigQuery), lit les 3 onglets produits en mémoire, normalise les colonnes et calcule les deltas ADDED/REMOVED par rapport au mois précédent avant d'écrire dans 4 tables BQ.
1
SCAN
Google Drive API — détection fichiers non ingérés
2
IDEMPOTENCE
SELECT DISTINCT catalog_date dans BQ — pas de doublons
3
LECTURE
3 onglets xlsx en mémoire (BytesIO) — Products, Approved, Certified
4
DELTAS
Calcul ADDED/REMOVED vs mois précédent BQ — clé composite
5
BQ WRITE
WRITE_APPEND dans 4 tables — autodetect schema
03 — Pipeline RM (Matières Premières)

Détection des ADDED/REMOVED par organisme certificateur

Fichiers CB → Comparaison BQ → Alertes email Cloud Scheduler · Mensuel
Chaque CB (organisme certificateur) dépose mensuellement son fichier Excel de matières premières sur Drive. La Cloud Function rm_ingest compare les RM du mois courant avec le mois précédent lu depuis BigQuery (architecture BQ-first, sans comparaison entre fichiers Drive). Elle vérifie les ADDED vs un référentiel historique validé, puis envoie un email récapitulatif SMTP aux destinataires configurés via Secret Manager.
1
IDEMPOTENCE
SELECT DISTINCT (month, cb_name) dans rm_events
2
SCAN DRIVE
1 fichier par CB — détection fichiers non ingérés
3
DELTAS
ADDED/REMOVED vs mois précédent BQ — onglet "Cosmos approved ingredients"
4
VALIDATION
Vérification ADDED vs rm_validated_history — statut submitted_and_validated
5
EMAIL
SMTP Gmail — App Password — destinataires via Secret Manager
