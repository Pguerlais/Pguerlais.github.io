01 — Concept

Qu'est-ce que WillItFlop ?

🎮
La problématique
Dans l'industrie du jeu vidéo, estimer la viabilité commerciale d'un titre avant sa sortie est un défi majeur. Des studios indépendants aux grands éditeurs, de nombreux jeux échouent malgré des budgets conséquents — faute d'une analyse de marché rigoureuse et de données exploitables en amont.

WillItFlop répond à cette question : « Si ce jeu vidéo existait, aurait-il des chances de réussir commercialement ? »
💡
La solution
Une plateforme web où l'utilisateur décrit un jeu vidéo fictif (genre, catégories, tags, prix…). Le système s'appuie sur 13 608 jeux Steam réels et un modèle XGBoost pour produire un score de viabilité commerciale, une probabilité de succès et un score Metacritic fictif.
🎯
La valeur ajoutée
Le projet combine collecte automatisée via 3 APIs (Steam, SteamSpy, Twitch), un pipeline end-to-end orchestré (Prefect + GitHub Actions + dbt) et de l'IA générative promptée (Groq + Pollinations) pour une expérience à la fois analytique et immersive.
02 — Pipeline & Automatisation

Une collecte de données entièrement automatisée

Boucle de collecte hebdomadaire Prefect Cloud · GitHub Actions
Un workflow GitHub Actions déclenché chaque lundi à 3h UTC via un webhook Prefect Cloud lance le pipeline ETL complet. Les nouvelles données sont récupérées, les données récentes (jeux < 1 an) rafraîchies, puis dbt transforme les données brutes en vue ml_features prête pour le modèle.
1
TRIGGER
Prefect Cloud schedule → webhook → GitHub Actions runner
2
COLLECTE
3 APIs : Steam Web API · SteamSpy · Twitch Helix
3
STOCKAGE
Insertion / mise à jour dans Supabase PostgreSQL
4
TRANSFORM
dbt modélise la vue ml_features (staging → ML → marts)
Réentraînement automatique du modèle GitHub Actions · CI/CD
Un second workflow surveille les fichiers ml/config.py et dbt/models/ml/**. Toute modification déclenche automatiquement le réentraînement XGBoost, puis les artefacts (model.pkl, preprocessor.pkl) sont commités automatiquement dans le dépôt, garantissant que l'API FastAPI utilise toujours le modèle le plus récent.
1
DÉTECTION
Push sur ml/ ou dbt/models/ml/ → workflow déclenché
2
ENTRAÎN.
XGBoost réentraîné sur la vue ml_features à jour
3
ARTEFACTS
model.pkl + preprocessor.pkl auto-commités dans le repo
4
PRÉDICTION
FastAPI sert le nouveau modèle via POST /predict sans intervention
03 — APIs de Collecte

3 sources de données · 13 608 jeux référencés

Steam Web API
Données produit
  • Détails du jeu : nom, type, catégories, date de sortie, description
  • Prix en euros
  • Top 5 avis positifs + 5 négatifs (filtrés par utilité)
  • Statistiques succès : nombre d'achievements, taux de déverrouillage médian
  • Présence Early Access, DLC, support multijoueur
SteamSpy API
Données marché
  • Estimation des propriétaires (min / max, midpoint calculé)
  • Peak CCU (joueurs simultanés max)
  • Temps de jeu médian
  • Tags attribués par les joueurs avec score de popularité
  • Liste paginée des jeux indie pour l'alimentation initiale
Twitch Helix API
Signal tendance
  • Recherche du jeu par nom dans le catalogue Twitch
  • Énumération de tous les streams actifs (jusqu'à 500, paginé)
  • Agrégation du nombre total de spectateurs en direct
  • Indicateur de présence Twitch (signal fort de succès : 52 % vs 18 %)
  • Rate limiting respecté (délais 300ms–1s entre appels)
04 — Machine Learning

Modèle XGBoost · Feature Engineering avancé

🤖
Modèle XGBoost
Classifieur binaire (succès / flop) entraîné sur 13 608 jeux Steam · 400 estimators · profondeur max 6 · learning rate 0,05 · validation croisée 5-fold stratifiée · scale_pos_weight calculé dynamiquement sur le déséquilibre réel des classes.
📈
Performances (CV)
ROC-AUC CV : 0,83 · F1 CV : 0,53 · Précision : 0,45 · Rappel : 0,64 · Seuil de décision F1-optimisé à 0,57 pour gérer le déséquilibre des classes (17 % de succès dans le dataset).
🎯
Feature Engineering
Prix log-transformé · tags répétés ×3, genres ×2, catégories ×2 dans la matrice pour pondérer leur influence via colsample_bytree · TF-IDF sur la description (200 features max, bigrams, sublinear_tf) · présence Twitch · Early Access · nombre de langues.
⚖️
Pondération par âge
Les jeux sortis depuis moins d'un an ont un poids réduit (0,05 → 1,0 selon l'âge) car leur label de succès est incertain. Cette décision améliore la robustesse du modèle sur les prédictions de jeux fictifs comparés à des titres récents.
05 — IA Générative

Prompts engineerés · Résultats personnalisés

✍️
Groq · Llama-3.3-70b
Génération d'une critique fictive de jeu en français (60 mots) avec nom de magazine inventé. Le prompt encode le verdict (Top/Flop), le score Metacritic fictif et les caractéristiques du jeu. Le ton est calibré dynamiquement selon le score (de "sévère" à "très enthousiaste") avec des guardrails : aucun vrai jeu ou magazine réel cité.
🎨
Pollinations · Flux
Génération d'une jaquette de jeu réaliste au format portrait, style packaging professionnel. Le prompt builder encode 80+ templates selon le genre, l'atmosphère, les mécaniques, le style visuel, la perspective et les modes de jeu — pour un résultat cohérent et professionnel à chaque génération.
🌐
DeepL API
Traduction automatique des descriptions Steam (en anglais) vers le français via l'endpoint POST /translate de l'API FastAPI. Permet d'afficher les données réelles des jeux comparables dans la langue de l'utilisateur, sans traitement NLP supplémentaire.

🎮 Tester le site ↗
Le livrable est une plateforme full-stack déployée (Vercel + Render + Supabase) avec un pipeline de données entièrement automatisé :

• Un verdict « Top! » / « Flop! » produit par le modèle XGBoost
• Une probabilité de succès (0–100 %) basée sur 13 608 jeux Steam réels
• Un score Metacritic fictif (40–95) projeté linéairement depuis la probabilité
• Une critique générée par IA (Groq / Llama-3.3-70b) avec ton calibré
• Une jaquette de jeu générée (Pollinations / Flux) depuis le formulaire
• Un dashboard analytique sur le marché indie (genres, prix, tags, timeline)

Architecture : Prefect + GitHub Actions pour l'orchestration · dbt + Supabase pour la donnée · XGBoost + FastAPI pour le ML · React + Vite en frontend.
